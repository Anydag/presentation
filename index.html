The benefits of Firestore

Lookup time ONLY dependable on amount of data retrieved
Automatically scales to match ANY usage
Automatic data syncronasitation across users
Offline support
Has ACID transactions
Where statements across multible fields

All data is indexed

- Zigzag merge join
Never loops over all data when searching

The drawbacks of Firestore

If it can't be index, it can't be searched
Mixing datatypes causes problems
Can't do where on: =!, OR, “%partial%”, a missing field
Cant do where < or> on one field and then order by a different field
